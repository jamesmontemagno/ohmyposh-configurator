## ğŸ” How to Use in Code

### TypeScript/React
```typescript
import { getIconById, searchIcons } from '@/constants/nerdFontIcons';

// Get a specific icon by ID
const plusIcon = getIconById('ui-plus');
console.log(plusIcon.char); // â†’ '+'

// Search for icons
const pythonIcons = searchIcons('python');
// â†’ [{ id: 'lang-python', name: 'Python', ... }]

// Get all icons in a category
import { getIconsByCategory } from '@/constants/nerdFontIcons';
const allLanguages = getIconsByCategory('Programming Languages');
```

### Using Icon Characters
```typescript
// Direct character usage in JSX
import { NERD_FONT_ICONS_MAP } from '@/constants/nerdFontIcons';

<span className="nerd-font-symbol">
  {NERD_FONT_ICONS_MAP['ui-plus'].char}
</span>

// Or with component
<NerdIcon icon="ui-plus" size={24} />
```

### Available Lookup Functions
- `getIconById(id: string)` - Get icon by unique ID
- `getIconByName(name: string)` - Get icon by display name
- `getIconsByCategory(category: string)` - Get all icons in a category
- `getCategories()` - Get list of all categories
- `searchIcons(query: string)` - Semantic search across all icon properties
- `getIconCharByAliasOrId(nameOrId: string)` - Get character directly

## ğŸ¯ Best Practices

1. **Use IDs over codes** - The unique ID system (`ui-plus`) is more readable than unicode codes
2. **Leverage aliases** - Search function supports multiple names for each icon
3. **Check for duplicates** - Use `searchIcons()` before adding new icons
4. **Verify rendering** - Test icons in the visual preview before shipping
5. **Document usage** - Add comments explaining non-obvious icon choices
6. **Keep consistent** - Follow the existing naming pattern for new icons

## ğŸ“ Contributing New Icons

To add new icons:

1. Find the icon in the [Nerd Fonts Visual Browser](https://nerdfonts.ytyng.com/)
2. Get the Unicode code point (in XXXX format)
3. Add to [src/constants/nerdFontIcons.ts](../src/constants/nerdFontIcons.ts)
4. Follow the existing format:
   ```typescript
   'category-name': { 
     id: 'category-name',
     name: 'Display Name',
     code: 'XXXX',  // 4-digit hex code
     char: '\uXXXX',
     category: 'Category Name',
     aliases: ['alias1', 'alias2'],
     description: 'What this icon represents'
   }
   ```
5. Update this documentation
6. Test and verify rendering

## ğŸ› Troubleshooting

### Icon not displaying
- **Check font loading:** Verify `index.css` includes the Symbols Nerd Font
- **Verify code point:** Use the visual browser to confirm the correct unicode
- **Browser support:** Ensure browser supports the unicode version needed

### Can't find an icon
- **Use semantic search:** `searchIcons('keyword')` searches names, descriptions, and aliases
- **Browse by category:** `getIconsByCategory('Your Category')` to browse
- **Check visual browser:** https://nerdfonts.ytyng.com/ to find the icon visually

### Unicode code mismatch
- **Verify in browser:** Open the visual browser and compare the code point
- **Check hex vs decimal:** Codes should be in hexadecimal (base 16)
- **Report issues:** Check GitHub issues if you find incorrect mappings
